Set Relations Application Assignment

Introduction

This assignment builds on the Functions application.  In the Functions program, the user entered two set rosters, and you counted all of the relations that were functions of various types.  In this assignment you are counting relations that meet the other criteria identified in the course material.  Recall, the 13 relation properties identified in CS 251 are

  1. Domain consumption
  2. Unique image
  3. Injection
  4. Surjection
  5. Reflexive
  6. Symmetric
  7. Transitive
  8. Not Reflexive
  9. Not Symmetric
  10. Not Transitive
  11. Anti Reflexive
  12. Anti Symmetric
  13. Anti Transitive

Using these properties, you can identify the following categories of relations:

  1. Functions, which satisfy (1) & (2)
  2. Bijective Functions, which satisfy (1), (2), (3), and (4)
  3. Equivalence relations, which satisfy (5), (6), and (7)
  4. Partial Order relations, which satisfy (5), (7), and (12)

Instructions

Write a C++ program that performs the following:

  1. Accept a list of elements on a single line (separated by spaces).  The elements are strings (with no embedded spaces), and there will be no more than 4 elements in this roster. This is set A.  Note that you must count the number of elements entered by the user on the line in order to determine the set size.  This set will never be empty.

  2. Accept a second list of  elements on a single line (separated by spaces). The elements are strings (with no embedded spaces), and there will be no more than 4 elements in this roster.  This is set B. Note that you must count the number of elements entered by the user on the line in order to determine the set size.  This set will never be empty.

  3. Display each of the following:
    1. The number of injective functions from A to B.  List all of the injective  functions in set notation.
    2. The number of equivalence relations on A.  List all of the equivalence relations in set notation.
    3. The number of partial order relations on B.  List all of the partial order relations in set notation.

Design Notes

Note the possible cardinalities: if A and B both have 4 elements, then the cross products will have 16 elements, which is a reasonable number.  The power set of the cross-product will have 216 = 65,536 elements, which is a larger number.  Note that of those 65,536 set relations, you will have at most  256 functions, 24 injective functions, and 15 equivalence relations (the counting details will become clear in the next module).

Try to avoid extensive testing of all relations - that can cause excessive runtimes for your program.  Before testing any of the 13 relation criteria, check set cardinality: any relation whose cardinality is less than the cardinality of the domain can not be a function, equivalence relation, or partial order relation since all elements of the domain must participate in these relations (due to domain consumption and reflexivity).
